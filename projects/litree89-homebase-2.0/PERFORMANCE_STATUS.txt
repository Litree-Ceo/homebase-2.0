🔧 TECHNICAL DETAILS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Root Cause Analysis:
    ms-azuretools.vscode-containers extension attempts to:
    • Monitor Docker daemon status
    • Connect to container runtime
    • Update container list on every file change
    
    When Docker is not responding or slow:
    • Extension hangs trying to connect
    • VS Code shutdown waits for extension (timeout)
    • Cascading EPIPE errors occur
    • Application becomes unresponsive

  Solution:
    • Disable the extension (not needed for development)
    • Docker/Dockerfile support still available via ms-azuretools.vscode-docker
    • Remote containers are enterprise feature (rarely used locally)
    • Cleanup removes hung processes and stale caches

  Why This Works:
    • Removes the problematic extension entirely
    • Cleans up hung processes
    • Removes accumulated Docker crud
    • Settings prevent similar extensions from hanging
    • Docker works fine without VSCode extension
